Записи

Пока что все типы, с которыми мы столкнулись, имеют значения, которые нельзя разложить: каждый экземпляр представляет собой отдельный фрагмент данных. Теперь мы собираемся увидеть наш первый класс составных типов: записи.

Записи позволяют составить значение из экземпляров других типов. Каждому из этих экземпляров будет присвоено имя. Пара, состоящая из имени и экземпляра определенного типа, называется полем или компонентом.

Объявление типа записи

Вот пример простого объявления записи:

type Date is record
   --  The following declarations are
   --  components of the record
   Day   : Integer range 1 .. 31;
   Month : Months;
   --  You can add custom constraints
   --  on fields
   Year  : Integer range 1 .. 3000;
end record;

Поля во многом похожи на объявления переменных, за исключением того, что они находятся внутри определения записи. Как и при объявлениях переменных, можно указать дополнительные ограничения при предоставлении подтипа поля.

type Date is record
   Day   : Integer range 1 .. 31;
   Month : Months := January;
   --  This component has a default value
   Year  : Integer range 1 .. 3000 := 2012;
   --                                 ^ Default value
end record;

Компоненты записи могут иметь значения по умолчанию. Когда объявлена переменная с типом записи, для поля с инициализацией по умолчанию будет автоматически установлено это значение. Значение может быть любым выражением типа компонента и может быть вычисляемым во время выполнения.

Агрегаты

Ada_Birthday    : Date := (10, December, 1815);
Leap_Day_2020   : Date := (Day   => 29,
                           Month => February,
                           Year  => 2020);
--                         ^ By name

Записи имеют удобную нотацию для выражения значений, показанную выше. Эта нотация называется агрегированной нотацией, а литералы - агрегатами. Их можно использовать в различных контекстах, которые мы увидим на протяжении всего курса, одним из которых является инициализация записей.

Агрегат - это список значений, разделенных запятыми и заключенных в круглые скобки. Это разрешено в любом контексте, где ожидается значение записи.

Значения для компонентов можно указать позиционно, как в примере Ada_Birthday, или по имени, как в Leap_Day_2020. Разрешено сочетание позиционных и именованных значений, но вы не можете использовать позиционную нотацию после именованной.

Выбор компонентов

Для доступа к компонентам экземпляра записи используется операция, называемая выбором компонентов. Это достигается с помощью точечной нотации. Например, если мы объявляем переменную Some_Day типа записи Date, упомянутого выше, мы можем получить доступ к компоненту Year, написав Some_Day.Year.

Рассмотрим пример:

with Ada.Text_IO; use Ada.Text_IO;

procedure Record_Selection is

   type Months is
     (January, February, March, April,
      May, June, July, August, September,
      October, November, December);

   type Date is record
      Day   : Integer range 1 .. 31;
      Month : Months;
      Year  : Integer range 1 .. 3000 := 2032;
   end record;

   procedure Display_Date (D : Date) is
   begin
      Put_Line ("Day:" & Integer'Image (D.Day)
                & ", Month: "
                & Months'Image (D.Month)
                & ", Year:"
                & Integer'Image (D.Year));
   end Display_Date;

   Some_Day : Date := (1, January, 2000);

begin
   Display_Date (Some_Day);

   Put_Line ("Changing year...");
   Some_Day.Year := 2001;

   Display_Date (Some_Day);
end Record_Selection;

Как вы можете видеть в этом примере, мы можем использовать точечную нотацию в выражении D.Year или Some_Day.Year. Где Year ‑ компонента для доступа к информации, хранящейся в этой переменной, а также для изменения этой информации в назначениях. Чтобы быть более конкретным, когда мы используем D.Year в вызове Put_Line, мы получаем информацию, хранящуюся в этом компоненте. Когда мы пишем Some_Day.Year := 2001, мы перезаписываем информацию, которая была ранее сохранена в компоненте Year в переменной Some_Day.

Переименование

В предыдущих главах мы обсуждали переименование подпрограммы и пакета. Мы также можем переименовывать компоненты записи. Вместо того, чтобы писать полный выбор компонентов с использованием точечной записи, мы можем объявить псевдоним, который позволит нам получить доступ к тому же компоненту. Это полезно, например, для упрощения реализации подпрограммы.

Мы можем переименовать компоненты записи, используя ключевое слово renames в объявлении переменной. Например:

Some_Day : Date
Y        : Integer renames Some_Day.Year;

Здесь Y является псевдонимом, так что каждый раз, когда мы используем Y ,мы действительно используем компонент Year в переменной Some_Day.

Давайте рассмотрим полный пример:

package Dates is

   type Months is
     (January, February, March, April,
      May, June, July, August, September,
      October, November, December);

   type Date is record
      Day   : Integer range 1 .. 31;
      Month : Months;
      Year  : Integer range 1 .. 3000 := 2032;
   end record;

   procedure Increase_Month (Some_Day : in out Date);

   procedure Display_Month (Some_Day : Date);

end Dates;

with Ada.Text_IO; use Ada.Text_IO;

package body Dates is

   procedure Increase_Month (Some_Day : in out Date) is
      --  Renaming components from
      --  the Date record
      M : Months  renames Some_Day.Month;
      Y : Integer renames Some_Day.Year;

      --  Renaming function (for Months
      --  enumeration)
      function Next (M : Months) return Months
        renames Months'Succ;
   begin
      if M = December then
         M := January;
         Y := Y + 1;
      else
         M := Next (M);
      end if;
   end Increase_Month;

   procedure Display_Month (Some_Day : Date) is
      --  Renaming components from
      --  the Date record
      M : Months  renames Some_Day.Month;
      Y : Integer renames Some_Day.Year;
   begin
      Put_Line ("Month: "
                & Months'Image (M)
                & ", Year:"
                & Integer'Image (Y));
   end Display_Month;

end Dates;

with Ada.Text_IO; use Ada.Text_IO;
with Dates;       use Dates;

procedure Main is
   D : Date := (1, January, 2000);
begin
   Display_Month (D);

   Put_Line ("Increasing month...");
   Increase_Month (D);

   Display_Month (D);
end Main;

Мы применяем переименование к двум компонентам записи Date в реализации процедуры Increase_Month. Затем вместо прямого использования Some_Day.Month и Some_Day.Year в следующих операциях мы просто используем переименованные версии M и Y.

Обратите внимание, что в приведенном выше примере мы также переименовали Months'Succ ‑ функцию, которая дает нам следующий месяц ‑ в Next.
